# Signal Finder + Codetape

**Emergency signal mapping · local stewardship · paper + wireless layers**

Signal Finder + Codetape is a commons seed document for helping communities locate surviving communication, need, and refuge signals after disaster without turning crisis into surveillance infrastructure.

**Find the signal. Protect the people.**

## Core idea

Signal Finder looks for wireless and visible signals. Codetape is the trust and handoff layer that keeps authority with a named local steward.

The system distinguishes infrastructure, beacon, tag, and need. It does not default to identifying people, uploading raw data, or exporting community maps to remote platforms.

## Layers

1. **Search** — drone, rover, kite, balloon, hand-carried node, bicycle mount
2. **Wireless** — Wi-Fi, cell, mesh, LoRa, Bluetooth, emergency SSIDs
3. **Beacon** — HELP_WATER, MEDICAL_NEED, SAFE_HOUSE, FOOD_HERE, CHARGING_STATION
4. **Paper** — tags, symbols, handwritten signs, local meanings, carried messages
5. **Steward** — clinic, library, school, fire station, mutual-aid hub, neighborhood council

## Process rhythm

1. **Find** — sense wireless and visible signals across the affected area.
2. **Read** — distinguish infrastructure, beacon, tag, and need. Not content. Not identity.
3. **Hand off** — return the map to the local steward. Not the cloud. Not a remote platform.
4. **Repair** — use the map to reconnect people, resources, and care.

## Responsible use covenant

This system does not:

- Conduct surveillance
- Upload to cloud by default
- Identify individuals unless rescue requires it
- Retain data post-crisis without consent
- Require an account, app, login, or platform to receive help
- Operate without a named local steward
- Export raw data without steward consent
- Operate outside emergency context without steward approval
